Using Position

Turning elements from stones into rhinos, aliens and flies.

You can use the css property ‘position’ to place elements exactly where you want them to go. Position can be used in the following ways:

When you use ‘position’ you can imagine you are turning your element into a stone, a rhino, an alien or a fly.

The Static Stone

Using position: static;

Imagine you have a field of long grass, when there is a stone, the grass grows around it. This is what it means to have the property position:static; . Every element is a stone by default.

e.g. We ask static block S to move 3px down from the top. But it doesn’t move. Because it is a stone and cannot hear us.

The Relative Rhino

Using position: relative;

Imagine you turn your object into a rhino. Unlike static stones, relative rhinos can move from their original position. When you tell the rhino to move, elements around the rhino must shift in order to make way for the rhino (or else they will be crushed). Because they are so heavy, rhinos also leave an empty space where they were initially standing.

e.g. We ask relative block R to move 3px down from the top of its original position. It moves for us! Note that the other elements do not move to fill the space it used to stand in because R is a rhino and has crushed its original position into mud. Nothing may grow there now.

The Absolute Alien

Using position: absolute;

Imagine you turn your object into a teleporting alien. Grass can grow underneath the absolute alien because the alien sits inside a spaceship that hovers. We can tell the alien to teleport anywhere relative to its mother ship The alien’s mother ship is the alien’s closest non-static ancestor (aliens do not recognize stones as ancestors because aliens are racist).

e.g. We ask absolute block A to move 10px across from the left. It teleports from its original position as if it were never there. It hovers 10px away from the left of its block R grandparent (notice that it completely ignores the existence of its static stone mother. Aliens are the worst like that.) We can define how high it flies using Z-index.

The Fixed Fly

Using position: fixed;

Imagine you turn your object into a fly. Then you take a piece of honey and smear it onto your browser window. The fly leaves its original location as if it were never there. It comes all the way up onto your computer screen to where you put the honey.

e.g. We ask fixed block F to move 50px across from the left. It remains at the top of the browser even when we scroll the webpage. We can make the z-index higher to ensure that F is right at the top, hovering over everything or make the z-index lower so that it slips beneath other elements as we scroll.
